# Vaste gegevens



## Vaste gegevens IMBOR-beheerobject
In IMBOR zijn een paar basiskenmerken verplicht bij elk beheerobject. Ook een element, bouwdeel of component zijn beheerobjecten in IMBOR. 

<figure>
<img src="../images/basiskenmerken-imbor-beheerobject.drawio.svg" alt="visuele weergave van de basiskenmerken">
<figcaption>De minimale onderdelen van de assetregistratie van een beheerobject in IMBOR. 
</figure>


1. Identificatie: Een unieke identifier voor het object
2. Domein: Het beheerdomein waartoe het object behoort (bijv. Wegen, Groen, Water, Kunstwerken)
3. Status: De status van het object (bijv. "in gebruik", "gepland", "buiten gebruik")

Daarnaast is impliciet altijd nodig:

4. Objecttype — het object moet een instantie zijn van een specifiek IMBOR-objecttype (bijv. imbor:Boom, imbor:Rioolput, imbor:Brug). Zonder typering kan IMBOR geen structuur bieden. 
5. Geometrie — IMBOR heeft een geometrie-addendum waarin per objecttype staat welke geometrie gewenst is. Eén geometrierepresentatie is verplicht, de rest optioneel.

> Hier moeten we het over hebben met de werkgroep: in de meeste beheersystemen is alleen op het niveau van het civiele kunstwerk een geometrie beschikbaar (meestal een 2D vlak);  Stelling: we geven een standaard decompositie mee, met de aanbeveling om van de kunstwerken een 3D representatie te hebben waarin deze delen apart gemodelleerd zijn. Dan hebben ze ook een geommetrie. Als een beheerder alleen over één geometrie beschikt, kunnen de overige onderdelen een punt-geometrie krijgen, daarmee is zeer duidelijk dat er nog geen geometrie beschikbaar is.
